> 文档定位：本文件是 `Task 2` 的审批前“思路与方法”闸口，用于说明后续如何求证，不构成终局结论，也不替代后续研究交付物。

> 当前状态：已吸收 `refine-stage-two-blackwell-hjt-model / Task 2` 的冻结版方法学更新；本文件现作为“继承上一轮方法学 + 本轮主线收束”的正式母本。

## 1. 使用边界与写法约束

- 本文件只整理“原 paper 口径”和“质疑对话口径”之间的差异，并说明后续准备采用的求证路径。
- 本文件保持中性审计式写法，不提前判断哪一侧正确，不把弱证据写成确定性事实。
- 本文件把后续证据来源区分为四类：`外部资料`、`工程参考`、`本地物理/工程推导`、`情景参数约束`。
- 本文件显式列出当前不能直接下结论的保留项，避免在审批前把待证问题误写成已证实结论。

| 模型分支 | 本轮经济模型只保留 `5年-GaAs`、`5年-HJT`、`10年-GaAs` 三条分支，禁止临时扩写第四条主分支 |
| 证据等级 | 统一标记为 `外部资料` / `工程参考` / `本地推导` / `情景约束` / `当前不可判定` |

### 2.1 本轮冻结版方法学补充

- `Rubin` 支线降级：上一轮“统一 `Blackwell` / 统一 `Rubin`”双链复算结果继续保留为证据底座，但本轮不再把 `Rubin` 作为主经济模型继续扩写；后续只保留其“说明差距可能缩小、但不足以直接翻转方向”的支线留痕作用。
- 统一 `Blackwell` 主线：本轮所有 BOM、采购价格、电源模型和三分支经济模型，统一回到 `Blackwell` 口径，不再混入新的 `Rubin` 经济计算。
- `GaAs/HJT` 边界：`GaAs` 代表当前较稳妥的空间电源主链，`HJT` 代表本轮允许并行比较的成熟路线；两者都必须在统一 `Blackwell` 主线下比较，不再另开“架构主线 × 电源路线”交叉组合。
- `5年/10年` 三分支：本轮只允许推进 `5年-GaAs`、`5年-HJT`、`10年-GaAs` 三条分支。其中 `5年` 分支需要显式写出“5 年服役后自动坠毁并重射一次”的口径，`10年` 分支当前仅保留 `GaAs` 版本。
- 当前不可直接下结论：`Rubin` 化后整星 BOM、寿命与辐射加固成本是否同步改善，`10年-HJT` 是否具备足够寿命/退化/采购链支撑，`70 kW/t` 是否对应整星总质量分母，`80-100颗/发` 能否升格为现实工程口径，以及“星链硅基太阳能电池”是否存在官方公开锚点，以上均继续保留为不可下结论项。

## 6. 单星质量拆解思路

### 6.1 拆解目标

- 目标不是立即证明 `2.14t` 或 `5-10t` 哪一项正确，而是先形成可复核的子系统拆解路径。
- 拆解对象至少包括：计算载荷、散热系统、光伏系统、电池系统、功率电子、平台结构、推进、姿轨控、通信/激光链路、热控与集成冗余。

### 6.2 拟采用的核算框架

```text
单星总质量
= 计算载荷质量
+ 散热系统质量
+ 光伏系统质量
+ 电池系统质量
+ 功率电子与配电质量
+ 平台与结构质量
+ 推进与工质质量
+ 通信/激光链路质量
+ 集成冗余与设计裕量
```

### 6.3 计划使用的求证路径

| 子项 | 拟采用方法 | 说明 |
|---|---|---|
| 计算载荷 | 从统一架构口径反推芯片数量、封装与机柜级集成需求 | 避免把地面整机参数直接映射为空间载荷 |
| 散热系统 | 根据持续热负荷和散热能力反推散热器面积与质量 | 需要与轨道热环境和辐射器性能联动 |
| 光伏与电池 | 由持续功率、昼夜周期和效率假设反推面积、储能和质量 | 该部分与第 6 节联动 |
| 平台/推进/通信 | 参考可比卫星平台和高功率卫星子系统经验区间 | 作为工程参考，不直接视为一手证据 |
| 冗余与裕量 | 设置保守/中性/成熟三档 | 用于吸收未公开设计细节的不确定性 |

### 6.4 当前不能直接下结论的点

- `70 kW/t` 一类指标是否适用于“整星总质量”而非“局部有效载荷质量”，当前仍需核对。
- 原 paper 的 `5-10t/颗` 是否包含较宽裕量、任务寿命冗余或非载荷附属系统，当前尚不能直接裁决。

## 7. 电池与光伏物理推导思路

### 7.1 推导目标

- 在统一持续功率目标下，给出电池储能需求、光伏面积、质量与成本的可追溯推导链。
- 把“空间电源系统”拆为 `光伏`、`电池`、`功率电子`、`结构与部署机构` 四类，而不是只给总项。

### 7.2 拟采用的物理推导链

```text
电池储能需求
= 持续功率需求 × 阴影段时长 / 可用放电深度 / 系统效率

光伏阵列输出需求
= （白昼段负载 + 为电池回充所需能量）/ 白昼段有效发电时长

光伏面积
= 阵列输出需求 / （轨道太阳常数 × 电池效率 × 姿态/温度/退化修正）

系统质量
= 光伏面积 × 单位面积质量
+ 电池储能 × 单位储能质量
+ 功率电子与结构附加质量
```

### 7.3 资料与参数来源分工

| 部分 | 主要依赖 | 说明 |
|---|---|---|
| 轨道昼夜周期 | `外部资料` | 轨道高度、倾角、食时比例等基础参数 |
| 光伏效率与退化 | `外部资料` + `工程参考` | 需区分 GaAs 与潜在硅基路线，并区分 `BOL/EOL` |
| 电池比能量、放电深度、循环寿命 | `外部资料` + `工程参考` | 需避免把地面商用品质参数直接套到空间级场景 |
| 面积、质量、成本复算 | `本地物理/工程推导` | 基于统一公式和情景参数区间计算 |
| 未来降本幅度 | `情景约束` | 只能做区间约束，不能直接写成已实现事实 |

### 7.4 当前保留项

- “星链硅基太阳能电池技术”是否存在足以支持成本替代判断的公开资料，当前仍需外部核查。
- GaAs 与潜在硅基路线即使都能找到资料，也不意味着两者在空间寿命、抗辐照、部署复杂度上可直接等价。

## 8. Blackwell / Rubin 统一复算思路

### 8.1 目标

- 上一轮已完成“统一 `Blackwell` / 统一 `Rubin`”双链复算；本轮仅继承其判别结果，不再把 `Rubin` 继续扩写为主经济模型。
- 解决“太空侧、地面侧分别引用不同芯片代际或不同系统口径”导致的比较偏差。
- 在方法学层面保留“统一 `Blackwell` / 统一 `Rubin`”的比较依据，但本轮执行层只允许基于统一 `Blackwell` 主线推进后续模型重建。

### 8.2 统一复算步骤

1. 逐项标注原 paper 和质疑对话当前使用的是 `Blackwell`、`Rubin` 还是过渡外推口径。
2. 固定比较分母，例如同一持续 IT 负载、同一时间窗、同一美元年份。
3. 构造“统一 `Blackwell`”复算链，太空侧和地面侧都只使用 `Blackwell` 对应参数。
4. 构造“统一 `Rubin`”复算链，太空侧和地面侧都只使用 `Rubin` 对应参数。
5. 比较两条复算链下的功耗、数量、质量、制造成本、发射成本和总拥有成本变化。
6. 仅在两条复算链都具备可追溯输入后，才讨论“结论是否翻转”。

### 8.3 拟输出的中间表

| 中间表 | 用途 |
|---|---|
| 架构映射表 | 标明每一侧当前使用的芯片代际、系统形态和引用位置 |
| 统一 `Blackwell` 复算表 | 在同一分母下复算太空侧与地面侧关键量 |
| 统一 `Rubin` 复算表 | 在同一分母下复算太空侧与地面侧关键量 |
| 翻转判定表 | 只记录“何处翻转、何处不翻转、何处当前不可判定” |

### 8.4 当前不能直接下结论的点

- 在尚未完成架构映射表之前，不能直接宣称“原 paper 结论已被 `Rubin` 时代推翻”。
- 若太空侧缺少对应代际的空间化约束参数，复算只能先做区间分析，不能伪装成精确结论。
