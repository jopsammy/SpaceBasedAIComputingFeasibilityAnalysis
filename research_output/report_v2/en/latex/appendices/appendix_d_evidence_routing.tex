\section{Evidence Chains, Citation Chains, and Script Routing}

This chapter serves as the executable traceability path from main-text conclusions back to original sources---the main text says ``see Appendix~D'', and readers come here to trace layer by layer via the tables.

\subsection{Five-Layer Routing Explanation}
\label{sec:five-layer}

All key conclusions in this report can be traced back to the most original public sources via the following five-layer routing:

\begin{enumerate}
  \item \textbf{Main-Text Conclusion} --- Root-level deliverable (\textit{Phase-2 Differential Analysis Preliminary Study.md}) or statistical figures in each results document
  \item \textbf{Cross-Verification Document} --- Per-Task gap closure status and cross-verification analysis (\path{stage2_gap_closure_verification.md}, etc.)
  \item \textbf{Structured JSON Input} --- Machine-readable parameter values (\path{stage2_mass_inputs_blackwell_hjt.json}, \path{stage2_cost_inputs_blackwell_hjt.json})
  \item \textbf{V Adjudication Records} --- V1--V37 per-parameter adjudication records in the internal parameter adjudication document (containing derivation chains, source URLs, per-parameter adjudication)
  \item \textbf{Original Source URL / Formula} --- Independently verifiable public web pages, DOIs, or physical formulas
\end{enumerate}

Usage: Find the conclusion you want to look up in the table $\to$ open the file in the corresponding column from left to right $\to$ each layer precisely points to more original supporting evidence. The rightmost column gives the final URL or formula, independently verifiable.\\[4pt]
\footnote{All Python calculation scripts and structured input JSON files referenced in this report will be open-sourced to a GitHub research repository upon finalization of the report, at which point readers can independently reproduce all model outputs. The current-stage script routing is in \S\ref{sec:script-entry}.}

\subsection{Python Script Entry Points}
\label{sec:script-entry}

All numerical calculations in this report are from the following two Python scripts. Both are independently Python-modeled, take structured JSON as input, and produce Markdown result documents; all values are reproducible and verifiable.

\subsubsection{stage2\_power\_system\_model.py}

\textbf{Function}: Orbital eclipse time calculation, battery capacity/mass, array area/mass, PCDU mass. Fixed Blackwell primary line, GaAs/HJT parallel, battery branched (DoD selected per branch).

\begin{itemize}
  \item \textbf{Input files}:\\
    \path{stage2_mass_inputs_blackwell_hjt.json}\\
    \path{stage2_cost_inputs_blackwell_hjt.json}
  \item \textbf{Output result file}:\\
    \path{stage2_power_system_results_blackwell_hjt.md}
  \item \textbf{Key formula chain}: Orbital period $\to$ eclipse time $\to$ eclipse-segment load energy $\to$ battery nominal capacity (/DoD/$\eta$)\\
    $\to$ recharge energy $\to$ sunlight-segment additional power $\to$ PCDU output $\to$ EOL array demand\\
    $\to$ array area (/$G$/$\eta_{\text{eol}}$/$f_{\text{pkg}}$)
  \item \textbf{Run command}: \path{python stage2_power_system_model.py}
\end{itemize}

\subsubsection{stage2\_mass\_cost\_model.py}

\textbf{Function}: Bottom-up calculation of payload mass, all-up satellite 9-subsystem mass/cost per-item summation, three-branch TCO (10-year total window, including manufacturing cost + launch cost + insurance + O\&M + decommissioning).

\begin{itemize}
  \item \textbf{Input files}:\\
    \path{stage2_mass_inputs_blackwell_hjt.json}\\
    \path{stage2_cost_inputs_blackwell_hjt.json}
  \item \textbf{Output result file}:\\
    \path{stage2_power_system_results_blackwell_hjt.md}
  \item \textbf{Key formula chain}: Compute payload mass $\gets$ \path{components[compute_payload].scenario_mass_kg} (direct read, no longer via kW/t denominator) $\to$ per-satellite mass = $\Sigma$(subsystem mass) $\to$ per-satellite manufacturing cost = $\Sigma$(subsystem cost) $\to$ total manufacturing cost = per-satellite cost $\times$ satellite equivalent $\times$ generations $\to$ total launch cost = per-satellite mass $\times$ satellite equivalent $\times$ launch unit price $\times$ generations $\to$ total cost = manufacturing + launch + insurance (8\%) + O\&M (\$5M/yr $\times$ 10) + decommissioning (\$3M/gen $\times$ generations)
  \item \textbf{Run command}: \path{python stage2_mass_cost_model.py}
\end{itemize}
