## 0. 结论先行

### 0.1 新 Bottom-Up 质量链结果

| 情景 | 计算载荷质量 | 推导路径 |
|---|---:|---|
| 公开硬件下界 | **~900-1,200 kg** | 150 颗 B200 级 GPU 裸模块 + 空间最低限度适配 |
| 基准工程估计 | **~1,400-1,800 kg** | 含 CPU 主机、NVLink 级互连、标准空间化增量 |
| 保守上界 | **~2,000-2,400 kg** | 含裕量、冗余、辐射加固上界 |

**核心结论**：新 bottom-up 链给出的基准区间（~1,400-1,800 kg）显著低于旧 kW/t 链的基准 2,727 kg，但上界 2,000-2,400 kg 接近旧链轻量化情景 2,143 kg。该差异的工程解释是：旧链的 `70 kW/t`（及本地衍生 `55 kW/t`）分母本身隐含了整星总质量中的非计算部分（平台、电源、热控等），不应单独用作计算载荷的比功率分母。

### 0.2 与旧 kW/t 链的对比

| 项目 | 旧链（kW/t 派生） | 新链（Bottom-Up） | 差异 |
|---|---:|---:|---|
| 轻量化 | 2,143 kg (70 kW/t) | ~900-1,200 kg | 旧链高出 ~80-140% |
| 基准 | 2,727 kg (55 kW/t) | ~1,400-1,800 kg | 旧链高出 ~50-95% |
| 保守 | 3,333 kg (45 kW/t) | ~2,000-2,400 kg | 旧链高出 ~40-65% |

### 0.3 关键依赖与不确定性

1. **GPU 单模块质量不可直接获取**：B200 SXM 模块的单独质量未被 NVIDIA 公开。本文通过 HGX B200 基板总质量（32 kg / 8 GPU，A1 级）和 GB300 计算托盘质量（29 kg / 4 GPU + 2 CPU，A1 级）反推。
2. **空间化增量是工程经验而非实测**：辐照屏蔽、热接口、结构封装的增量来自 LEO 卫星工程的通用经验范围（1.2x-2.0x），不是 AI1 的实测数据。
3. **Grace CPU 必要性质疑**：若 AI1 采用不同于 GB200 Superchip 的 CPU-GPU 架构（如独立 x86 主机或完全无 CPU 的嵌入式推理架构），CPU 质量估算需相应调整。

## 2. 地面硬件质量拆解

### 2.1 数据锚点总表

| 锚点 | 值 | 证据等级 | 含义 |
|---|---:|---:|---|
| HGX B200 基板 | **32 kg** (8 GPU) | A1 | 8 颗 B200 SXM + 2 颗 NVSwitch + PCB + 热接口 |
| GB300 NVL72 计算托盘 | **29 kg** (4 GPU + 2 CPU) | A1 | 含液冷冷板、存储、BMC、风扇 |
| GB200/GB300 NVL72 整机柜 | **~1,360 kg** (72 GPU, ~120 kW) | B1 | 含机柜结构、供配电、液冷歧管、线缆 |
| DGX B200 整机 | **142.4 kg** (8 GPU) | A1 | 含机箱、6 PSU、2 CPU、存储、风扇 |
| Supermicro 4U B200 系统 | **107 kg** (8 GPU) | B1 | 液冷版，含机箱、4 PSU、2 CPU |

### 2.2 计算核心质量推导

#### 2.2.1 从 HGX B200 基板反推单 GPU 模块质量

```
HGX B200 基板 (32 kg) 包含：
  - 8 × B200 SXM GPU 模块
  - 2 × NVSwitch 芯片
  - 基板 PCB
  - 热接口组件（冷板安装座、TIM 等）

单 GPU SXM 模块质量估算范围：
  下界：~2.0 kg（仅 GPU die + HBM stack + 封装基板 + SXM 连接器）
  基准：~2.8 kg（含 NVSwitch 分摊和 PCB 分摊后每 GPU 等效）
  上界：~3.5 kg（保守假设，含更多分摊）

依据：
  - 32 kg / 8 = 4.0 kg/GPU（包含全部基板分摊）
  - 扣除 PCB (~4 kg)、NVSwitch (~2 kg)、热接口 (~2 kg)，剩余 ~24 kg 为 GPU 模块
  - 24 kg / 8 = 3.0 kg/GPU（去除基板分摊后的 GPU 模块）
  - H100 SXM5 模块尺寸约 110mm × 70mm，估重约 1.5-2.0 kg
  - B200 双 die + 192GB HBM3e（8 层堆栈）比 H100 更重，估计 +40-60%
```

#### 2.2.2 从 GB300 计算托盘反推

```
GB300 NVL72 计算托盘 (29 kg) 包含：
  - 4 × B300 SXM GPU 模块（每颗 288GB HBM3e, 12 层堆栈, ~1,200-1,400W TDP）
  - 2 × Grace CPU 模块（每颗 72 核 ARM Neoverse V2, 480GB LPDDR5X）
  - 液冷冷板（GPU + CPU + ConnectX-8）
  - PCB 主板
  - 8 × E1.S SSD + 1 × M.2 SSD
  - BMC 管理模块
  - 小风扇（气冷组件）
  - 连接器与机械框架

估算分解：
  - B300 GPU 模块: 4 × ~3.5 kg = ~14 kg  (288GB HBM3e 12-hi 显著重于 B200)
  - Grace CPU 模块: 2 × ~1.0 kg = ~2 kg
  - 液冷冷板组件: ~3 kg
  - PCB + 连接器: ~4 kg
  - 存储: ~1 kg
  - BMC + 风扇: ~1 kg
  - 机械框架: ~2 kg
  - 其他（线缆、ConnectX-8 等）: ~2 kg
  合计: ~29 kg ✓

单 B300 GPU 模块（去液冷、去存储、去管理）质量: ~3.5 kg
单 B200 GPU 模块（192GB HBM3e, 8-hi）估重: ~2.5-3.0 kg（HBM 堆栈少 4 层）
```

#### 2.2.3 选定 GPU 质量基线

本文以 B200 级 GPU（1,000W TDP）作为地面参考基线，原因：
- B200 公开资料更充分（NVIDIA 官方 PCF 数据表，A1 级）
- 150 kW / 1,000W = **150 颗 GPU** 的计算更直观
- 若实际采用 B300（~1,200-1,400W），GPU 颗数减少（~107-125 颗），但单颗质量增加，总核心质量变化可控

**选定值：单 B200 级 GPU SXM 模块 = 2.5-3.0 kg**

### 2.3 地面计算硬件质量汇总（150 GPU 规模）

| 组件 | 单件质量 | 数量 | 合计 | 说明 |
|---:|---:|---:|---:|
| B200 级 GPU SXM 模块 | 2.5-3.0 kg | 150 | **375-450 kg** | GPU die + HBM + 封装 + 连接器 |
| Grace 级 ARM 主机 CPU | 0.8-1.2 kg | 75 | **60-90 kg** | 若沿用 2 GPU : 1 CPU 的 Superchip 比例 |
| NVLink 级互连背板 | — | 1 套 | **120-200 kg** | 空间级高速背板/交换芯片（等效于 NVSwitch） |
| 计算子板 PCB 与连接器 | — | 若干 | **80-150 kg** | GPU-CPU 载板、板间互联 |
| **地面计算硬件核心小计** | | | **635-890 kg** | 去液冷、去机柜、去供配电的"裸计算"硬件 |

### 2.4 地面冷却与供配电质量（将从空间版移除的部分）

基于 GB300 NVL72 机柜分解（总重 ~1,360 kg, 72 GPU），需要从空间版中移除以下地面专属项目：

| 地面专属项 | NVL72 中估重 | 150 GPU 等效 | 说明 |
|---:|---:|---:|
| 液冷系统（歧管、CDU、管路） | ~200-300 kg | ~420-625 kg | 按 GPU 比例线性放大；空间版用辐射散热替代 |
| 19" 机柜结构 | ~150-200 kg | ~150 kg | 机柜结构不随 GPU 数量线性增长 |
| AC 供配电（8 × 电源架） | ~240-300 kg | ~300-400 kg | 空间版由光伏阵列直接供电，经 PCDU 调节 |
| 线缆与配电背板 | ~80-120 kg | ~170-250 kg | 按比例估计 |

**要点**：上述组件在空间版中完全不存在（由光伏阵列、PCDU、散热板、平台结构等取代，这些已在现有 BOM 中作为独立大件计入）。因此**不应将地面机柜总质量直接等比缩放到空间计算载荷**。

## 3. 空间化增量

### 3.1 增量分解

空间版计算载荷需要在"裸计算硬件"基础上增加以下项目：

| 增量项 | 质量因子 | 150 GPU 估计 | 依据 |
|---:|---:|---:|
| **辐照屏蔽** | 裸电子质量的 +15-30% | **~150-350 kg** | LEO ~600 km 轨道的 TID 环境；多层复合屏蔽（Al + Ta/W）比纯铝屏蔽效率高 53-72%（PMC 2021）；若采用 COTS GPU + 局部屏蔽策略则处于上界 |
| **空间热接口** | 裸电子质量的 +10-20% | **~100-250 kg** | 替代地面液冷冷板的热管/导热索/热扩展板；需将 ~150 kW 热负荷传导至散热板 |
| **结构封装与承力** | 裸电子质量的 +15-25% | **~150-300 kg** | 发射振动载荷（通常 8-12 g_rms 随机振动 + 冲击）；真空环境下的约束与支撑；展开/锁紧机构 |
| **空间级连接器与线缆** | +5-10% | **~50-120 kg** | 真空兼容连接器、低放气材料、屏蔽线缆 |
| **冗余与交叉备份** | +10-20% | **~100-250 kg** | 空间不可维修性要求；N+M 冗余 GPU 和 CPU 模块 |

### 3.2 空间化总乘数

```
空间化增量合计：裸计算硬件的 +55% 到 +105%

空间计算载荷 = 地面计算硬件核心 × (1.55 ~ 2.05)
             = (635 ~ 890) × (1.55 ~ 2.05)
```

### 3.3 三层质量链结果

| 层级 | 质量范围 | 推导 |
|---:|---|
| **下界（公开硬件 + 最低空间适配）** | **~900-1,200 kg** | 取 GPU 模块质量的低值 + 空间化增量的低端 |
| **基准（工程估计）** | **~1,400-1,800 kg** | 取 GPU 模块质量的中值 + 空间化增量的中端 |
| **上界（保守 + 冗余）** | **~2,000-2,400 kg** | 取 GPU 模块质量的高值 + 空间化增量的高端 |

### 3.4 公式表述

```
单星计算载荷质量 (kg) =
    N_GPU × m_GPU                                      # GPU 裸模块
  + N_CPU × m_CPU                                      # 主机 CPU 模块
  + m_interconnect                                     # 互连背板/交换
  + (上述小计) × f_shielding                            # 辐照屏蔽系数 (0.15-0.30)
  + (上述小计) × f_thermal_iface                        # 空间热接口系数 (0.10-0.20)
  + (上述小计) × f_structure                            # 结构封装系数 (0.15-0.25)
  + (上述小计) × f_redundancy                           # 冗余系数 (0.10-0.20)

其中：
  N_GPU = 150（基于 150 kW / 1,000W per B200）
  m_GPU = 2.5-3.0 kg/颗（基于 HGX B200 32 kg / 8 GPU 和 GB300 托盘 29 kg 反推）
  N_CPU = 75（基于 GB200 Superchip 2GPU:1CPU 架构假设）
  m_CPU = 0.8-1.2 kg/颗（ARM 服务器 CPU 封装典型量级）
  m_interconnect = 120-200 kg（空间级高速背板/交换芯片估计）
```

## 4. 新旧链对比与废弃说明

### 4.1 旧 kW/t 链的废弃理由

| 废弃项 | 理由 |
|---|---|
| `70 kW/t` 分母 | 来源为二手媒体转述（Inkl/Tom's Hardware, B1 级），非 NVIDIA 或 SpaceX 官方规格表。更关键的是，该比率究竟是"整星级功率密度"还是"载荷级功率密度"未明确。若理解为整星级（含平台、电源、热控），则它不应被用作纯计算载荷的比功率。 |
| `55 kW/t` 本地衍生值 | 从 `70 kW/t` 向下衍生而来，没有任何独立外部来源支持。这是典型的"二次推导 + 本地调整"，违反本轮"不得把本地反算写成主链基线"的原则。 |
| `45 kW/t` 保守值 | 同上，纯本地情景参数。 |
| `2,727 kg = 150 kW / 55 kW/t` | 分子有二手交叉一致支撑（150 kW peak），但分母缺乏独立验证，且分母的定义域（整星 vs 载荷）存在根本性歧义。 |
| 循环引用风险 | 旧链中的"计算载荷质量"通过 kW/t 推导后又进入平台系数和总质量，而 kW/t 本身又可能是从总质量反算出来的，构成潜在循环引用。 |

### 4.2 新链的不确定性说明

| 不确定来源 | 影响方向 | 量级 |
|---|---|---|
| GPU 单模块质量不可直接获得 | 可能低估或高估 20-30% | ±200-300 kg |
| 若 AI1 使用 B300（更重但更少颗数） | 核心质量变化可控（更重×更少 ≈ 持平） | ±100 kg |
| 若 AI1 不使用 Grace CPU 架构 | CPU 质量可调减 0-90 kg | -0-90 kg |
| 空间化增量系数来自通用工程经验 | 实际增量取决于 AI1 的具体设计 | ±200-400 kg |
| 150 kW 功率的组成（纯 GPU vs 含主机） | 若含主机功耗则 GPU 颗数减少 | -20-50 颗 GPU, -50-150 kg |

### 4.3 新旧链定位对比

| 维度 | 旧链 (kW/t) | 新链 (Bottom-Up) |
|---|---|---|
| 推导方向 | 自上而下（整星功率密度 → 反推质量） | 自下而上（地面硬件质量 → 叠加空间增量） |
| 核心锚点 | `70 kW/t`（二手转述, B1） | HGX B200 32 kg（NVIDIA 官方, A1） |
| 可复核性 | 可复核公式，但分母来源不可复核 | 每个锚点均可追溯到公开 URL |
| 不确定性表达 | 三点情景（45/55/70 kW/t），但分母变动本身无物理依据 | 双界表达（下界/上界），每个增量项有独立物理含义 |
| 适用层级 | 整星级公式（有分母定义域歧义） | 载荷级（仅计算硬件 + 计算专属空间适配） |

## 5. 未闭合项

### 5.1 当前仍无法确定的项

| 条目 | 状态 | 阻塞原因 | 对质量链的影响 |
|---|---|---|---|
| B200/B300 SXM 单模块精确质量 | 未闭合 | NVIDIA 未公开单模块质量 | 影响核心质量 ±20-30% |
| AI1 实际 GPU 型号与颗数 | 未闭合 | 无一手规格表 | 影响 GPU 颗数和单颗功率假设 |
| AI1 的 CPU-GPU 架构 | 未闭合 | 未知是否沿用 Grace CPU 或采用其他方案 | 影响 CPU 质量估算 0-90 kg |
| 空间化增量系数 | 不可精确闭合 | 增量取决于 AI1 的具体工程方案（屏蔽策略、热设计、结构设计），这些在当前阶段不可知 | 影响空间化总质量 ±200-400 kg |
| AI1 的轨道精确参数 | 未闭合 | 仅知约 600 km 圆形轨道（二手转述） | 影响辐射屏蔽需求（600 km 处于内范艾伦带下缘，质子通量对倾角敏感） |
| GPU 在空间的降额/降频策略 | 未闭合 | GPU 在真空中可能需要降低 TDP 以保证可靠性 | 若降额则单星需更多 GPU 颗数，核心质量增加 |

### 5.2 后续可推进方向

1. 若能获取 NVIDIA 的 B200 SXM 模块运输/包装规格（含质量），可将核心质量锚点升级为 A1 级。
2. 若能确认 AI1 的实际架构（是否基于 GB200 Superchip 或全定制方案），可大幅缩小 CPU 和互连质量的不确定性。
3. 若能从 SpaceX FCC/ITU 申报文件中提取 AI1 的更多工程参数（质量、功率、尺寸），可直接替代当前的底部推导。
