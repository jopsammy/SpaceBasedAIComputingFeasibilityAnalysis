## 1. DoD 边界修正

### 1.1 错误性质

上一轮文档将 **DoD（Depth of Discharge，放电深度）** 表述为与 GaAs/HJT 光伏路线并列的评价维度，或将其置于光伏路线比较的语境中讨论。DoD 的正确系统边界是：

> **DoD 是电池储能子系统的独立参数，决定电池名义容量 `E_battery = P_eclipse * t_eclipse / (DoD * η_discharge)`，从而影响电池质量与成本。DoD 与 GaAs/HJT 的关系不是"直接评价光伏路线"，而是通过储能约束间接耦合——光伏阵列需要满足电池回充需求，但 DoD 本身不随光伏路线变化。**

### 1.4 修正后的统一口径

- **DoD 是电池储能子系统的参数**，用于从食区负载能量推导电池名义容量。
- **GaAs 和 HJT 共享同一 DoD 值**，不是因为两条路线都"选择"了 0.40，而是因为电池子系统独立于光伏路线——同一颗卫星的电池配置不因光伏电池化学体系改变。
- **DoD 的乐观/基准/保守取值**仅影响电池容量、电池质量、电池成本，不改变光伏阵列面积或光伏质量（除非 DoD 过低导致电池质量激增、进而通过平台系数放大整星质量——此为间接耦合，见第 2 节）。
- 任何文档不得将 DoD 写为"光伏路线的评价维度"或"路线选择参数"。

## 2. 电源系统耦合物理链

### 2.1 完整公式链

电源系统五个子系统之间的物理耦合如下。**这不是五个孤立子系统，而是通过功率流、能量流和质量流耦合的单一电源系统。**

```
                    ┌──────────────────────────────────────────────────────┐
                    │                  轨道力学输入                          │
                    │  h = 600 km, β = 0, R_earth = 6378.137 km            │
                    │  T_orbit = 2π√((R+h)³/μ)                             │
                    │  t_eclipse = T_orbit × asin(R/(R+h))/π               │
                    │  t_sunlight = T_orbit - t_eclipse                     │
                    └─────────┬────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                       1. 电池 (Battery)                                    │
│  ┌──────────────────────────────────────────────────────────────┐       │
│  │ E_load_eclipse = P_sustained × t_eclipse          [kWh]      │       │
│  │ E_battery_nominal = E_load_eclipse / (DoD × η_discharge)     │       │
│  │                                         [kWh]                 │       │
│  │ M_battery = E_battery_nominal × 1000 / ρ_specific   [kg]     │       │
│  │ C_battery = E_battery_nominal × unit_cost          [USD]     │       │
│  └──────────────────────────────────────────────────────────────┘       │
│  输出 → E_recharge（回充能量需求）→ 2. 光伏阵列                            │
│  输出 → M_battery, C_battery → 5. 平台/整星                                │
└─────────────────────────────┬───────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                       2. 光伏阵列 (Array)                                  │
│  ┌──────────────────────────────────────────────────────────────┐       │
│  │ E_recharge = E_load_eclipse / (η_charge × η_discharge)       │       │
│  │ P_recharge_extra = E_recharge / t_sunlight          [kW]     │       │
│  │ P_pcdu_out = P_sustained + P_recharge_extra         [kW]     │       │
│  │ P_array_eol = P_pcdu_out / η_bus                    [kW]     │       │
│  │ A_array = P_array_eol × 1000 / (G × η_eol × f_pkg) [m²]     │       │
│  │ M_array = A_array × σ_array                         [kg]     │       │
│  │ C_array = P_array_bol × 1000 × unit_cost            [USD]    │       │
│  └──────────────────────────────────────────────────────────────┘       │
│  输入 ← E_recharge（来自电池子系统）                                        │
│  输出 → P_array_eol → 3. PCDU                                             │
│  输出 → M_array, C_array → 5. 平台/整星                                    │
└─────────────────────────────┬───────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                       3. PCDU（功率电子/配电）                              │
│  ┌──────────────────────────────────────────────────────────────┐       │
│  │ P_pcdu_rating = max(P_array_eol, P_peak)           [kW]      │       │
│  │ M_pcdu = P_pcdu_rating / ρ_pcdu                    [kg]      │       │
│  │ C_pcdu = P_pcdu_rating × unit_cost                 [USD]     │       │
│  └──────────────────────────────────────────────────────────────┘       │
│  输入 ← P_array_eol（来自光伏子系统）                                       │
│  输出 → M_pcdu, C_pcdu → 5. 平台/整星                                      │
│  输出 → 母线功率 → 4. 散热（间接，散热面积由峰值热负荷驱动）                   │
└─────────────────────────────┬───────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                       4. 散热 (Thermal)                                    │
│  ┌──────────────────────────────────────────────────────────────┐       │
│  │ A_radiator = P_peak / (εσT⁴)                      [m²]      │       │
│  │ M_thermal = A_radiator × σ_thermal                 [kg]      │       │
│  │  注：散热面积由峰值热负荷驱动（含计算载荷废热），                 │       │

│  ┌──────────────────────────────────────────────────────────────┐       │

│  │ M_subtotal = M_payload + M_thermal + M_power + ...            │       │
│  │ M_platform = M_subtotal × ratio_platform            [kg]      │       │

│  输入 ← M_array, M_battery, M_pcdu, M_thermal（来自各子系统）               │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 关键公式

#### 电池容量公式（核心）

```
E_battery_nominal = (P_sustained × t_eclipse) / (DoD × η_discharge)

其中：
  P_sustained = 120 kW           # 持续负载功率（二档）
  t_eclipse   = 35.4 min         # 食时（一档，600 km / β=0 轨道力学导出）
  DoD         = 0.40 (基准)      # 放电深度（三档，电池子系统独立参数）
  η_discharge = 0.95             # 放电效率（二档，行业通用值）
```

DoD 在分母位置，因此电池容量对 DoD 高度敏感：
- DoD = 0.60 → E_battery = 124.5 kWh → M_battery = 655 kg（轻量化情景）
- DoD = 0.40 → E_battery = 186.8 kWh → M_battery = 983 kg（基准情景）
- DoD = 0.20 → E_battery = 373.6 kWh → M_battery = 1966 kg（保守情景）

#### 质量耦合公式

```
M_power = M_array + M_battery + M_pcdu

其中分量公式：
  M_array   = A_array × σ_array
  M_battery = E_battery_nominal × 1000 / ρ_battery
            = (P_sustained × t_eclipse × 1000) / (DoD × η_discharge × ρ_battery)
  M_pcdu    = max(P_array_eol, P_peak) / ρ_pcdu
```

**耦合特征**：
- `M_battery = f(DoD, ρ_battery)`：DoD 在分母，电池比能 ρ_battery 在分母——两者同等地位决定电池质量。
- `M_array = f(η_eol, σ_array, E_recharge)`：光伏效率决定面积，但回充需求 E_recharge 来自电池子系统的充电效率——这是跨子系统耦合。
- `M_pcdu = f(P_array_eol, P_peak, ρ_pcdu)`：额定功率来自光伏阵列输出——跨子系统耦合。

#### 成本耦合公式

```
C_power = C_array + C_battery + C_pcdu

其中分量公式：
  C_array   = P_array_bol × 1000 × unit_cost_array    [USD]
  C_battery = E_battery_nominal × unit_cost_battery    [USD]
  C_pcdu    = P_pcdu_rating × unit_cost_pcdu           [USD]
```

#### 间接耦合路径

```
DoD ↓ → E_battery ↑ → M_battery ↑ → M_power ↑ → M_subtotal ↑ → M_platform ↑ → M_总 ↑ → C_发射 ↑
                   → C_battery ↑ → C_power ↑
DoD ↑ → E_battery ↓ → 反向
```

此路径说明：虽然 DoD 不是光伏路线评价参数，但 DoD 通过质量链间接影响整星总质量和发射成本——这是电池子系统对整星的"穿透性影响"，不是 DoD 直接评价光伏路线的证据。

### 2.3 子系统间输入输出关系

| 流向 | 源子系统 | 目标子系统 | 传递量 | 耦合类型 |
|---|---|---|---|---|
| ① | 电池 → 光伏 | Battery | Array | `E_recharge`（回充能量） | 能量耦合：电池充电效率决定光伏日照段额外功率 |
| ② | 光伏 → PCDU | Array | PCDU | `P_array_eol`（EOL 阵列功率） | 功率耦合：PCDU 额定功率由光伏输出决定 |
| ③ | PCDU → 散热 | PCDU | Thermal | 母线效率损失 → 热负荷 | 热耦合：PCDU 效率损失贡献散热需求 |
| ④ | 电池 → 平台 | Battery | Platform | `M_battery` | 质量耦合：电池质量进入整星质量求和 |
| ⑤ | 光伏 → 平台 | Array | Platform | `M_array` | 质量耦合：光伏质量进入整星质量求和 |
| ⑥ | PCDU → 平台 | PCDU | Platform | `M_pcdu` | 质量耦合：PCDU 质量进入整星质量求和 |
| ⑦ | DoD → 电池 → 光伏 → PCDU | Battery → Array → PCDU | | 链式耦合：DoD 影响电池容量（①），进而影响回充功率（②），进而影响 PCDU 额定功率（③）——但当前模型下 P_array_eol 的实际变化幅度远小于 P_peak（150 kW），因此 PCDU 额定功率通常由 P_peak 主导，而非光伏回充功率。 |

### 3.4 场景交叉约束

| 约束 | 说明 |
|---|---|
| DoD 与电池比能不可同时取极端乐观 | 高比能电池通常循环寿命较短，而高 DoD 加速容量退化——乐观情景的 0.60 DoD + 220 Wh/kg 组合隐含"高循环应力 + 高比能短寿命"的物理矛盾风险（当前模型未建模 DoD 与循环寿命的 trade-off） |
| DoD 与光伏面积间接耦合 | DoD 影响电池容量 → 影响回充能量 → 影响光伏阵列面积。但此耦合路径的实际幅度较小：DoD=0.60 vs 0.20 导致的回充功率变化约 1-2 kW 级别，而光伏阵列总面积由 ~150 kW 级别主导 |
| GaAs 保守情景的面密度与 EOL 退化不应同时取极端 | 保守情景已取 EOL=0.28（高退化），再取面密度=2.5 kg/m² 会双重放大保守度——当前保守情景取面密度=1.8，仅 EOL 取保守 |

## 4. 公式敏感性 vs 证据差异分离

### 4.1 问题描述

当前文档中将"参数变化导致的输出区间"归因于单一口径，实际存在两类性质完全不同的区间来源：

- **公式敏感性**：参数在公式中的数学位置（如在分母、指数位）决定了输出对该参数的敏感程度。这是结构性的、数学的，与证据强弱无关。
- **证据差异**：参数的置信区间宽度由底层证据的强度决定。有公开锚点的参数区间窄，无锚点的参数区间宽。

两者混合表述会导致读者无法判断：一个宽区间是因为"这个参数真的很敏感"还是"我们真的不知道这个参数该取多少"。

### 4.2 公式敏感性来源（结构性）

| 参数 | 在公式中的位置 | 敏感性特征 | 数值示例（基准 → 该参数翻倍/减半） |
|---|---|---|---|
| **DoD** | 分母（电池容量 = 负载能量 / DoD） | **高**：DoD 从 0.40 → 0.20 时电池容量翻倍（186.8 → 373.6 kWh） | DoD=0.40→0.20：电池质量从 983 kg → 1966 kg（+100%），电源总质量从 1942 kg → 3268 kg（+68%） |
| **电池比能 ρ_battery** | 分母（电池质量 = 容量 / 比能） | **高**：比能从 190 → 155 Wh/kg 时电池质量增加 23% | ρ=190→155：电池质量从 983 kg → 1205 kg（+222 kg），正比于 1/ρ |
| **光伏 EOL 效率 η_eol** | 分母（阵列面积 = 功率 / (G × η_eol)） | **高**：η_eol 从 0.29 → 0.14 (HJT) 时面积放大 2.07x | η=0.29→0.14：阵列面积从 625 m² → 1295 m²（+107%），阵列质量从 938 kg → 1554 kg（+66%） |
| **阵列面密度 σ_array** | 乘子（阵列质量 = 面积 × 面密度） | **中**：面密度从 1.5 → 1.8 kg/m² 时阵列质量增加 20% | σ=1.5→1.8：阵列质量从 938 kg → 1125 kg（+187 kg） |
| **PCDU 比功率 ρ_pcdu** | 分母（PCDU 质量 = 功率 / 比功率） | **低**：PCDU 质量本身很小（21 kg），即使比功率减半也只增加 ~21 kg 绝对值 | ρ=10→5 kW/kg：PCDU 质量从 21 kg → 42 kg（+21 kg），仅影响电源总质量 1% |
| **母线效率 η_bus** | 分母（阵列功率 = PCDU 功率 / η_bus） | **低**：η_bus 从 0.94 → 0.90 时阵列功率增加 4.4%，面积同比例增加 | η=0.94→0.90：阵列面积从 625 m² → 653 m²（+28 m², +4.5%） |
| **散热面密度 σ_thermal** | 乘子（散热质量 = 面积 × 面密度） | **中低**：面密度从 5.0 → 6.5 kg/m² 时散热质量增加 165 kg | 对电源总质量无直接影响（散热质量不进入 M_power），仅影响整星总质量 |

**核心发现**：电源系统质量的最大公式敏感性来源是 **DoD**（分母位置，线性放大）和 **光伏 EOL 效率**（分母位置，线性放大）。这两者的区间宽度对电源总质量的影响远大于其他参数。

### 4.3 证据差异来源（认知性）

| 参数 | 证据等级 | 公开锚点强度 | 区间跨度 | 区间宽的原因 |
|---|---|---|---|---|
| **电池比能 ρ_battery** | 三档 | 强：公开产品级锚点（ICP25 155 Wh/kg, BAT600-P3）+ 行业范围 | 155-220 Wh/kg（1.42x） | 区间窄：公开产品锚点有效约束了下限和上限 |
| **GaAs BOL 效率** | 二档 | 强：AzurSpace 4G32 官方产品页 32% class | 0.32-0.36（1.13x） | 区间窄：厂商公开数据表锚定准确值 |
| **GaAs 面密度 σ_array** | 三档 | 中：行业参考区间 1.5-2.5 kg/m²，无 AI1 一手规格 | 1.3-1.8（1.38x） | 区间中等：行业共同知识约束了合理范围 |
| **DoD** | 三档 | 中：行业经验范围 20-40% + ICE/Satsearch 产品锚点 | 0.20-0.60（3x） | 区间宽：公式敏感性（分母）+ DoD 与循环寿命的 trade-off 使极端值合理 |
| **HJT 面密度 σ_array** | 三档 | 中弱：依赖 33FG Research 分析而非一手规格 | 1.0-3.0（3x） | 区间宽：HJT 空间封装的精确面密度取决于展开机构类型，公开证据不足以缩窄 |
| **PCDU 比功率 ρ_pcdu** | 四档 | **极弱**：无任何公开空间级 PCDU 质量或比功率数据 | 7.5-12 kW/kg（1.6x） | 区间看似不宽，但实际不确定性可能更大——1.6x 是工程占位的人为缩窄，实际可能达 2-20 kW/kg（一个数量级） |
| **阵列封装修正因子 f_pkg** | 三档 | 弱：工程通用值 0.85，无产品锚点 | 当前固定 0.85（1x） | 区间未表达不确定性——f_pkg 涵盖布线、盖玻片、温度、姿态和局部遮挡，合理范围可能 0.80-0.90 |
| **散热面密度 σ_thermal** | 四档 | 弱：航天器热控结构面密度范围 3-8 kg/m²，无 AI1 一手 | 4.0-6.5（1.63x） | 区间中等但仍有较大不确定性 |

### 4.4 混合项的拆分

以下参数同时受公式敏感性和证据差异影响，需要分离表述：

| 参数 | 公式敏感性贡献 | 证据差异贡献 | 拆分说明 |
|---|---|---|---|
| **电池质量 M_battery** | DoD 在分母（结构性的 3x 放大：0.20 vs 0.60） | 电池比能 ρ_battery 的证据区间 155-220 Wh/kg（1.42x） | 电池质量从 565 kg（轻量化）到 2071 kg（保守）约 3.7x 跨度：其中约 3x 来自 DoD 公式敏感性，约 1.42x 来自比能证据差异——**跨度的主导来源是公式敏感性而非证据不足** |
| **阵列质量 M_array (GaAs)** | EOL 效率在分母 → 面积变化（EOL 从 0.33 → 0.28 = 1.18x 面积放大） | 面密度 σ_array 的证据区间 1.3-1.8 kg/m²（1.38x） | 阵列质量从 713 kg（轻量化）到 1165 kg（保守）约 1.63x 跨度：BOL/EOL 效率变化贡献约 1.18x，面密度变化贡献约 1.38x——**跨度的主导来源是面密度的证据差异** |
| **电源总质量 M_power (GaAs)** | DoD（电池容量）+ EOL 效率（阵列面积）串联 | 比能 + 面密度 + PCDU 比功率的证据差异叠加 | 从 1297 kg（轻量化）到 3268 kg（保守）约 2.52x 跨度：DoD 使电池质量 3x，但电池仅占总质量的 30-63%，加权后对总质量的放大约 1.68x——**公式敏感性与证据差异的贡献大致相当** |

### 4.5 关键分离结论

1. **DoD 的宽区间（0.20-0.60 = 3x）主要是公式结构性的**：DoD 在分母位置，这种结构性敏感无论证据多强都会存在。即使 AI1 公开了精确 DoD=0.40，DoD 作为"调节电池质量 vs 循环寿命"的设计杠杆仍需要多情景表达。

2. **PCDU 的窄区间（7.5-12 kW/kg = 1.6x）不代表证据强**：这个"窄"是工程占位的人为缩窄——因为没有任何公开锚点，无法确定真实的上界和下界。实际不确定性可能远超 1.6x。

3. **电池比能的窄区间（155-220 Wh/kg = 1.42x）代表证据相对可靠**：公开产品锚点有效约束了合理范围。

4. **HJT 面密度的宽区间（1.0-3.0 kg/m² = 3x）代表证据不足**：柔性/刚性封装的路径分叉导致面密度可能跨越数倍——这不是公式敏感性，而是真的不知道 AI1 用什么封装方案。

5. **禁止混合表述**：后续文档中，当说明某个输出区间"宽"时，必须分别指出是 "公式结构使然（如 DoD 在分母）" 还是 "证据不足使然（如 PCDU 比功率无锚点）"。
